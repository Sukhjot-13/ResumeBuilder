# Audit Report — Resume Builder Codebase
> Generated: 2026-05-24

## Framework & Tech Stack
- **Framework**: Next.js 16.0.7 (App Router), React 19.2.0
- **Auth**: Custom JWT-based auth (JOSE for Edge middleware, jsonwebtoken for Node) with OTP email login (Brevo)
- **Database**: MongoDB via Mongoose (8.19.3)
- **Styling**: Tailwind CSS v4
- **UI Components**: Custom (no shadcn/ui)

## Auth
- **Custom JWT auth** with OTP-based email login (Brevo)
- Access tokens (15min) + refresh tokens (15 days) with rotation
- Middleware proxy (`src/proxy.js`) reads cookies → verifies tokens → injects `x-user-id` header
- Auth context (`AuthContext.js`) on client side calls `/api/user/profile` to check auth state
- Role-based permission system (ADMIN=0, DEVELOPER=70, SUBSCRIBER=99, USER=100)
- Credit tracking (Free: 2/day, Pro: 200/month)

## Database
- **MongoDB** (not Supabase/Postgres)
- **6 models**: User, Resume, ResumeMetadata, RefreshToken, Transaction, Plan

## Resume Generation
- **File**: `src/services/contentGenerationService.js` → calls `geminiService.js`
- **Model**: Gemini Flash (`gemini-flash-latest`)
- **Prompt strategy**: `src/lib/promptConfig.js` (3 tiers: basic/standard/premium per role)
- **AI Edit**: `src/services/aiResumeEditorService.js` — edit resume with natural language via Gemini
- **Resume Parsing**: `src/services/resumeParsingService.js` — upload PDF/DOCX → Gemini extracts structured data
- **Schema source of truth**: `src/lib/resumeFields.js` → auto-generates Mongoose schemas, AI prompts, and form state

## PDF Generation
- **Library**: `@react-pdf/renderer` (v4.3.1)
- **Endpoint**: `POST /api/render-pdf-react` — takes resumeData + template name, renders via react-pdf to blob
- **Templates** (6): ClassicTemplate, ClassicTemplate2, Creative, Modern, Professional, Simple
- **Renderer**: `PdfResumeRenderer.js` — generic wrapper, passes data to template component

## Auth Middleware
- **Proxy**: `src/proxy.js` (Edge middleware) — checks tokens from cookies, injects `x-user-id` header into API requests, handles token rotation, subscription checks, route protection
- **Server Actions**: `src/lib/serverAuth.js` — reads cookies, calls `verifyAuth` for server-side auth

## Key Observations
1. **MongoDB** is the database (not PostgreSQL/Supabase as the spec assumes)
2. **Gemini** (Google) is the AI provider (not OpenAI/Anthropic)
3. **Auth is custom JWT** (no NextAuth.js)
4. **No Supabase** — no storage for PDFs, no RLS, no real-time subscriptions
5. **No Redis/queue system** — everything is synchronous request-response
6. **`/api/profile`** reads/writes a local `data.json` file (separate from user profile in MongoDB)
7. **Puppeteer** and **jsPDF** are installed as dependencies but unused
8. **No API key auth system** currently exists for external services
9. **No rate limiting** beyond the credit system
