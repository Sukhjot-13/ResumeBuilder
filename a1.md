# Audit Report ŌĆö Resume Builder Codebase
> Generated: 2026-05-24 | Last updated: 2026-05-24

## Framework & Tech Stack
- **Framework**: Next.js 16.0.7 (App Router), React 19.2.0
- **Auth**: Custom JWT-based auth (JOSE for Edge middleware, jsonwebtoken for Node) with OTP email login (Brevo)
- **Database**: MongoDB via Mongoose (8.19.3)
- **AI**: Gemini Flash (`gemini-flash-latest`) via unified AI config at `src/lib/ai/`
- **PDF**: `@react-pdf/renderer` v4.3.1
- **Styling**: Tailwind CSS v4
- **UI Components**: Custom (no shadcn/ui)

## Auth
- Custom JWT auth with OTP-based email login (Brevo)
- Access tokens (15min) + refresh tokens (15 days) with rotation
- Middleware proxy (`src/proxy.js`) reads cookies ŌåÆ verifies tokens ŌåÆ injects `x-user-id` header
- Auth context (`AuthContext.js`) on client side calls `/api/user/profile` to check auth state
- Role-based permission system (ADMIN=0, DEVELOPER=70, SUBSCRIBER=99, USER=100)
- Credit tracking (Free: 2/day, Pro: 200/month)

## Database
- **MongoDB** (not Supabase/Postgres)
- **5 models**: User, Resume, ResumeMetadata, RefreshToken, Transaction

## Resume Generation
- **Flow**: API route ŌåÆ `resume-generator.js` ŌåÆ `callAI('RESUME_GENERATION')` ŌåÆ Gemini Flash
- **Prompt strategy**: `src/lib/promptConfig.js` (3 tiers: basic/standard/premium per role)
- **AI Edit**: `aiResumeEditorService.js` ŌåÆ `callAI('AI_EDIT')` ŌåÆ Gemini Flash
- **Resume Parsing**: `resumeParsingService.js` ŌåÆ `callAI('RESUME_PARSING')` ŌåÆ Gemini Flash
- **Gatekeeper**: `callAI('GATEKEEPER')` ŌåÆ Gemini Flash (evaluates job listings for automation)
- **Schema source of truth**: `src/lib/resumeFields.js` ŌåÆ auto-generates Mongoose schemas, AI prompts, and form state

## PDF Generation
- **Library**: `@react-pdf/renderer` (v4.3.1)
- **Flow**: API route ŌåÆ `pdf-generator.js` ŌåÆ react-pdf ŌåÆ Buffer
- **Templates** (6): ClassicTemplate, ClassicTemplate2, Creative, Modern, Professional, Simple

## Key Architecture Decisions (Session 2 ŌĆö Job Automation)

### Database
- **All automation data in MongoDB** ŌĆö no Supabase/Postgres. Models: `ApiKey`, `JobListing`, `GatekeeperDecision`, `Application`, `PlatformSession`, `JobCriteria`, `SchedulerSettings`, `GatekeeperRules`, `DailyCount`.

### Dashboard Placement
- **Same Next.js app at `/automation/*`** ŌĆö not a separate deploy. Reuses existing custom JWT auth, MongoDB connection, and deployment.
- **Future split is straightforward**:
  - Copy `/automation/*` pages + API routes ŌåÆ new Next.js app
  - Copy Mongoose models + DB connection utility ŌåÆ both apps
  - Copy auth utilities (`AuthContext`, `requirePermission`, `jwt.js`) ŌåÆ new app
  - Add `MONGODB_URI` + `JWT_SECRET` to new app's env vars (same MongoDB, same JWT secret)
  - CORS headers only needed on Resume Builder routes called by the new Dashboard
  - No auth rewrite ŌĆö stateless JWT works on both apps with the same secret

### Gatekeeper AI
- **Gemini Flash initially** via existing `callAI('GATEKEEPER')` in `src/lib/ai/config.js`. Worker calls it through `POST /api/gatekeeper/evaluate` API route (HTTP), not directly.
- **Swappable** ŌĆö already in `config.js`, change via `AI_TASK_GATEKEEPER=provider:model` env var.

### Worker ŌåÆ App Authentication
- **API Keys** ŌĆö `ApiKey` Mongoose model + Bearer token middleware. Future-proof for when they're separate services.

### Templates
- **Exposed via `GET /api/resume/templates`** ŌĆö returns `{ id, name, ... }` for each template. Worker discovers templates dynamically.

### Resume Modes (future)
- **Mode A**: Custom resume per job ŌĆö uses job description + profile ŌåÆ `callAI('RESUME_GENERATION')`
- **Mode B**: Same resume for all ŌĆö uses pre-set profile, no tailoring
- **On-the-fly generation** ŌĆö no PDFs stored in DB; generate per application
